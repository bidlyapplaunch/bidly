## P0 ‚Äî CRITICAL (correctness, data integrity, security regressions)

### Winner processing & auction end (the highest-risk cluster)
- **BACKEND-01 / SVC-03 / SVC-05** ‚Äî There are **two** end-of-auction systems running simultaneously: the 5s poller in `server.js:1064-1164` (picks winner as `bidHistory[last]`, sends `sendAuctionWonNotification`) **and** `winnerProcessingService` (picks winner by sorted amount, sends `sendWinnerNotification`). They can pick different winners and **both** send email. Worse, `auctionEndService.js` is a **third, broken** path wired into `processEndedAuctions` that calls a non-existent `getShopifyService().getClient()` and throws on every call.
  **Fix:** One service owns auction-end. Atomically claim the auction (`findOneAndUpdate({_id, winnerProcessed:false}, {$set:{winnerProcessingLock:true}})`) before any side effect. Delete `auctionEndService.js` and the duplicate email block in `server.js`. Use `determineWinner` (sorted) as the single source of truth.
- **BACKEND-08** ‚Äî `completedAt` is set by `auctionEndService`/queried by `processEndedAuctions`, but **not declared in `models/Auction.js`** (there's even an index on the phantom field). `strict:true` drops it on save ‚Üí ended auctions never marked processed ‚Üí **infinite reprocessing**.
  **Fix:** Add `completedAt: { type: Date, default: null }` to the schema; verify it persists.
- **BACKEND-07** ‚Äî `buyNow` (`auctionController.js:1170-1198`) does 3 non-atomic saves (`active` ‚Üí addBid ‚Üí `ended`). Race-prone; concurrent Buy-Now/bid can double-win.
  **Fix:** Single guarded `findOneAndUpdate` like `placeBid` already uses.
- **BACKEND-14** ‚Äî `placeBid` re-`save()`s the doc after its atomic update to apply popcorn extension (`auctionController.js:913-924`), reintroducing a race.
  **Fix:** Apply the extension via `findOneAndUpdate` with `$max` on `endTime`, not a full save.

### Live security holes (NEW ‚Äî not in AUDIT.md)
- **SVC-01** ‚Äî `POST /api/billing/override {plan:"enterprise"}` is mounted with only `identifyStore` (no auth). **Free Enterprise for anyone.** Same primitive in `debugRoutes.js:206` `/set-plan`. ‚Üí require `requireAuth` on all billing mutations.
- **SVC-02** ‚Äî `debugRoutes` mounted unconditionally in prod (`server.js:701`). Exposes token previews/scopes per store, `DELETE /clear-store/:shop` (force-uninstall any store), `/set-plan`. ‚Üí gate behind `NODE_ENV !== 'production'` + auth.
- **SVC-08** ‚Äî `POST /api/customers/:id/bid` has **no** `verifyCustomerToken` (every sibling route does). Anyone can forge bid history / inflate `auctionsWon`. ‚Üí add token check + owner match.
- **SVC-09** ‚Äî App Proxy storefront bid/buy-now endpoints (`appProxyRoutes.js`) **don't validate the Shopify proxy HMAC** and trust `logged_in_customer_id` from the query string ‚Üí bid as any customer. ‚Üí verify proxy signature, derive identity from verified session.
- **SVC-23** ‚Äî `analyticsRoutes` has no caller auth (`optionalStoreIdentification` only) ‚Üí any shop's revenue/user analytics readable via `?shop=`. ‚Üí add `requireAuth`.
- **SVC-16** ‚Äî `ensureCustomer` is read-then-write with no upsert ‚Üí concurrent first-bids race into `E11000`. ‚Üí `findOneAndUpdate(..., {upsert:true})`.

### Admin / platform
- **ADMIN-02** ‚Äî `onboardingComplete: ... || true` (`auction-admin/src/App.jsx:203`) is always truthy ‚Üí onboarding never shows. ‚Üí use `?? false`.
- **ADMIN-01** ‚Äî `main.jsx` ships 5 `originalLog('üöÄ‚Ä¶')` calls that **bypass** the debug gate (fire in prod), with imports interleaved with executable statements. ‚Üí remove logs, hoist imports.
- **PLAT-01** ‚Äî `@react-router/serve` (used by `start`) is **not in `dependencies`**; `--omit=dev` prod install may not have it ‚Üí boot crash. ‚Üí add to `dependencies`, verify clean-container boot.

## P1 ‚Äî HIGH (performance & reliability)

### Backend hot path & queries
- **BACKEND-03** ‚Äî `getDailyStats` issues up to **365 sequential `find()`** for a 1y range. ‚Üí one `$group` aggregation bucketed by day.
- **BACKEND-04** ‚Äî `getAnalytics` loads **every auction** (full `bidHistory`/`productData`, no `.lean()`) into memory to count statuses. ‚Üí aggregation + projection.
- **BACKEND-05** ‚Äî `getAuctionStats` loads all auctions and recomputes status ~13√ó/auction; also includes soft-deleted. ‚Üí compute once / aggregate, add `isDeleted:{$ne:true}`.
- **BACKEND-06** ‚Äî Every bid does `io.to(room).emit` **and** a global `io.emit` of the full decorated auction to **all** connected sockets ‚Üí fan-out storm + cross-tenant data leak. ‚Üí drop the global emits; scope to rooms, slim payload.
- **SVC-06** ‚Äî `POST .../auction-metafields` loops 10 metafields √ó (GET + PUT) = **~20 serial REST calls per bid**, blocking the bid response and burning the 40-call bucket. ‚Üí single `metafieldsSet` GraphQL mutation.
- **BACKEND-12** ‚Äî Auction mutations make the server **HTTP-call itself** (`fetch ${API_BASE_URL}/api/metafields/...`), re-entering Express/auth/store-lookup per bid. ‚Üí extract to a service, call in-process.
- **BACKEND-11** ‚Äî `refreshAllProductData` does `Auction.find({})` across **all tenants** with serial Shopify calls in a request handler ‚Üí timeout + multi-tenancy violation. ‚Üí scope by shop, batch, background job.
- **SVC-07** ‚Äî `shopifyGraphQLService.executeGraphQL` has **no 429/THROTTLED retry** (unlike `shopifyService.executeWithRetry`). Duplication/winner-product creation fails hard under load. ‚Üí add backoff.

### Front-end performance
- **CUST-01** ‚Äî Widget polls every 5s even with socket connected (the guard checks `socket.connected` synchronously right after `io()`). ‚Üí start polling only on `connect_error`/timeout, clear on `connect`.
- **CUST-04 / PERF-001** ‚Äî Per-card `setInterval` countdowns: 50 auctions √ó up to 3 timers = 150 timers/sec ‚Üí mobile jank. ‚Üí one shared ticker context.
- **CUST-06** ‚Äî After every bid the widget calls full `fetchVisibleAuctions()` (spinner + re-render all cards) although the bid response + socket event already carry new state. ‚Üí update just the affected auction.
- **CUST-07** ‚Äî `joinAuction` effect depends on `[auctions]` ‚Üí re-emits `join-auction` for all auctions on every bid/poll. ‚Üí track joined IDs in a ref.
- **ADMIN-03** ‚Äî Dashboard socket effect keyed on `[i18n]` ‚Üí tears down/re-registers listeners + `fetchStats()` storm on every locale/render change. ‚Üí `[]`-keyed connection effect, read i18n via ref.
- **ADMIN-12** ‚Äî `getShopFromURL()` (70-line multi-source resolver) runs **inside the request interceptor on every API call**. ‚Üí memoize per session.
- **SVC-10** ‚Äî Blast email `sendAllAtOnce` sends up to 10k emails in a tight loop, fire-and-forget with no `.catch` ‚Üí SMTP ban + possible unhandledRejection crash. ‚Üí throttle + `.catch`.
- **SVC-15** ‚Äî `emailService` does sync `fs.existsSync`/`readFileSync` per template per email (blocking the event loop, repeated per recipient). ‚Üí cache localized templates.
- **PLAT-02** ‚Äî Root `build` compiles the unrelated standalone `auction-admin` via `buildWithSkip.js` ‚Üí bloated deploys; an admin build failure aborts the Remix deploy. ‚Üí decouple.

### Reliability
- **SVC-11** ‚Äî `processAllEndedAuctions` returns `undefined` ‚Üí cron always logs "nothing processed" and masks whether the safety net works. ‚Üí return a count.
- **SVC-18** ‚Äî Winner processing releases the lock on error but never advances state ‚Üí cron **retries the same failing auction forever** (e.g. deleted product ‚Üí 404). ‚Üí add `processingAttempts` + `failed` terminal state.
- **SVC-14** ‚Äî `emailService.sendEmail` swallows SMTP errors (returns `{success:false}`, ignored by callers) ‚Üí failed winner/invoice emails are invisible. ‚Üí surface/throw + retry.
- **BACKEND-02** ‚Äî `optionalAuth` calls undefined `verifyToken` (`middleware/auth.js:127`); the throw is swallowed ‚Üí auth silently degrades on `customizationSettings`/`emailSettings` routes. ‚Üí use `jwt.verify`.
- **PLAT-04** ‚Äî `api.auctions.stats.jsx:3` still falls back to a hardcoded backend URL (siblings throw). ‚Üí throw-on-missing.
- **PLAT-06/07** ‚Äî `useAuthenticatedFetch` returns a new fn every render ‚Üí unstable identity drives effect/`useCallback` loops (`AuctionDetails`, `AuctionForm`). ‚Üí `useMemo(() => app ? authenticatedFetch(app) : fetch, [app])`.
